# E2E Run вҖ” Execute Tests + Error Classification + Report

Run existing E2E tests, classify failures, and generate dual reports.

All output must be in **з№Ғй«”дёӯж–Ү**.

## Step 1: Locate Spec File

- If `$ARGUMENTS` provided, use it as the spec path
- If no argument, look for spec files in `tests/e2e/` (excluding `pages/`, `auth/`, `fixtures/`) and ask the user which to run
- If the spec file does not exist, suggest `/e2e:analyze` вҶ’ `/e2e:plan` вҶ’ `/e2e:create` instead

## Step 2: Locate Skill Directory & Load References (MANDATORY)

**Step 2a: Find the skill directory**
Locate the e2e-testing skill directory by finding its SKILL.md:
1. `Glob("**/e2e-testing/SKILL.md")` вҖ” searches CWD (works during plugin development)
2. If not found: `Glob("**/e2e-testing/SKILL.md", path: "~/.claude/plugins")` вҖ” searches plugin cache

Extract the **directory path** from the result (remove `/SKILL.md` suffix). This is `$SKILL_DIR`.

**Step 2b: Read references**
Read the following files using `$SKILL_DIR/references/{filename}`:
- `$SKILL_DIR/references/error-discrimination.md` вҖ” error classification framework
- `$SKILL_DIR/references/report-template.md` вҖ” report format template

Do NOT proceed without reading all listed files. If both Glob attempts fail, report the error and stop.

## Step 3: Verify Dev Server

Check if `http://localhost:5173` is reachable. If not:
- Try `cd app && pnpm dev` in background, or
- Inform the user that the dev server needs to be running

## Step 4: Execute Tests

Run from the `app/` directory:
```bash
E2E_REPORT_NAME={page-name} pnpm test:e2e -- {spec-path}
```

## Step 5: Error Discrimination

IF any test fails:
в”ңв”Җв”Җ Classify each failure using error-discrimination.md:
в”Ӯ   в”ңв”Җв”Җ FORM SUBMISSION error?
в”Ӯ   в”Ӯ   в”ңв”Җв”Җ Environment keyword (disabled/archived/locked/suspended)? вҶ’ ENVIRONMENT: report data state issue, suggest UI fix
в”Ӯ   в”Ӯ   в”ңв”Җв”Җ Recoverable keyword (йҮҚиӨҮ/duplicate/invalid format)? вҶ’ Log as RECOVERABLE
в”Ӯ   в”Ӯ   в””в”Җв”Җ Other вҶ’ Log as NON-RECOVERABLE
в”Ӯ   в”ңв”Җв”Җ PAGE LOADING error вҶ’ NON-RECOVERABLE
в”Ӯ   в””в”Җв”Җ ELEMENT INTERACTION error вҶ’ NON-RECOVERABLE
в””в”Җв”Җ Do NOT auto-fix вҖ” only classify and report

## Step 6: Generate Dual Reports

1. **HTML report** вҖ” Already generated by Playwright at `playwright/reports/{page-name}/`
2. **Markdown report** вҖ” Write `playwright/reports/{page-name}/test-report.md` per `references/report-template.md`, including:
   - Test results summary (pass/fail/skip counts)
   - Per-failure error classification
   - Recommended actions for each failure type
