# E2E Run — Execute Tests + Error Classification + Report

Run existing E2E tests, classify failures, and generate dual reports.

All output must be in **繁體中文**.

## Step 1: Locate Spec File

- If `$ARGUMENTS` provided, use it as the spec path
- If no argument, look for spec files in `tests/e2e/` (excluding `pages/`, `auth/`, `fixtures/`) and ask the user which to run
- If the spec file does not exist, suggest `/e2e:analyze` → `/e2e:plan` → `/e2e:create` instead

## Step 2: Load References (MANDATORY — do not skip)

Read these files before proceeding:
- `references/error-discrimination.md` — error classification framework
- `references/report-template.md` — report format template

Do NOT proceed to Step 3 without reading both files.

## Step 3: Verify Dev Server

Check if `http://localhost:5173` is reachable. If not:
- Try `cd app && pnpm dev` in background, or
- Inform the user that the dev server needs to be running

## Step 4: Execute Tests

Run from the `app/` directory:
```bash
E2E_REPORT_NAME={page-name} pnpm test:e2e -- {spec-path}
```

## Step 5: Error Discrimination

IF any test fails:
├── Classify each failure using error-discrimination.md:
│   ├── FORM SUBMISSION + recoverable keyword (重複/duplicate/invalid format)?
│   │   └── YES → Log as RECOVERABLE (do NOT auto-fix in Run mode)
│   │   └── NO → Log as NON-RECOVERABLE
│   ├── PAGE LOADING error → NON-RECOVERABLE
│   └── ELEMENT INTERACTION error → NON-RECOVERABLE
└── Do NOT auto-fix — only classify and report

## Step 6: Generate Dual Reports

1. **HTML report** — Already generated by Playwright at `playwright/reports/{page-name}/`
2. **Markdown report** — Write `playwright/{page-name}/test-report.md` per `references/report-template.md`, including:
   - Test results summary (pass/fail/skip counts)
   - Per-failure error classification
   - Recommended actions for each failure type

## Step 7: Completion

Tell the user:

```
✅ 測試執行完畢。
📄 報告：
   - HTML: playwright/reports/{page-name}/
   - MD: playwright/{page-name}/test-report.md
```

If there are failures:
```
⚠️ 失敗項目已分類為可恢復/不可恢復，詳見報告。
```
